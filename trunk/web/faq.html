Here are the frequently asked questions concerning boson


Will there be a port to gtk (or any other toolkit) ?

Even in the lowest level, boson code makes heavy use of Qt. So it's quite hard to make a generic RTS core engine that is toolkit agnostic.

I have plans to make the server Qt-independant, using only standart C component, and no GUI (graphical user interface), so that the server may be used on any unix-like computer.


What does Boson stand for ?

As far as I know, a boson is a sub-particle, like for example an electron. But since boson.org, boson.net and boson.de are already registered, we are trying to find a new name for this game. So please make any suggestions to the boson mailing list.


What Qt / KDE version do I need ?

The current CVS version and therefore the next release version depends on QT3/KDE3. Make also sure to install libkdegames from the kdegames package.


Is there a CVS access ?

Yes you can access CVS through SourceForge. See the project page for more infos.
You can use
cvs -d:pserver:anonymous@cvs.boson.sourceforge.net:/cvsroot/boson login
cvs -z3 -d:pserver:anonymous@cvs.boson.sourceforge.net:/cvsroot/boson co modulename
(where modulname is one of code and data) to download the current code and data.

Note that you need both, code and data to use boson. You also have to link the admin/ dir from code to data:
cd data ; ln -s ../code/admin admin

Now you can do the (for KDE CVS) usual
make -f Makefile.cvs ; ./configure ; make ; make install
in both directories.
