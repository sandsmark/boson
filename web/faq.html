Here are the frequently asked questions concerning boson


Will there be a port to gtk (or any other toolkit) ?
Even in the lowest level, boson code make a heavy use of Qt. So it's quite hard to make a generic RTS core engine that is toolkit agnostic.

I have plans to make the server Qt-independant, using only standart C component, and no GUI (graphical user interface), so that the server may be used on any unix-like computer.


What does Boson stand for ?
As far as I know, a boson is a sub-particle. Well... the name was chosen in a hurry, and I don't have time to find a better one. Feel free to make any proposal to the boson mailing list.

By the way, we are looking for a new name, as www.boson.org has been stolen by a bad guy who wants us to pay 500$.



What about Qt 2 / KDE 2 ?
Have a look at the roadmap, it's planned for the next release 0.5 From then on, all boson version will need QT2.1 and KDE 2.0 libraries.
